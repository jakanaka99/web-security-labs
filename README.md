# Web Security Labs

This repository documents my learning journey in **Web Application Security and Penetration Testing**.

The notes here are based on hands-on labs from **PortSwigger Web Security Academy**, where I practice identifying and exploiting common web vulnerabilities.

## Purpose

The goal of this repository is to:

- Document vulnerabilities I encounter while solving labs
- Understand how different web attacks work
- Record payloads and testing methods
- Build a structured reference for web security concepts

## Lab Documentation Format

Each lab follows a structured format:

Observation  
Understanding the application behavior and identifying potential vulnerabilities.

Test  
The approach used to verify whether the application is vulnerable.

Payload  
The crafted input used to exploit the vulnerability.

Result  
The outcome of the attack and confirmation of the vulnerability.

Impact (when applicable)  
Explanation of the security risk and what an attacker could achieve.

## Topics Covered

This repository will include labs related to:

- Path Traversal
- Access Control
- Authentication
- SQL Injection
- Cross-Site Scripting (XSS)
- File Upload Vulnerabilities
- OS Command Injection
- Server-Side Request Forgery (SSRF)

## Tools Used

- Burp Suite
- Browser Developer Tools
